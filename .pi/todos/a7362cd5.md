Implemented the SongQueueDO RpcTarget-handle design end-to-end for application callers.

Completed:
- Added `apps/api/src/lib/rpc-result.ts` for transport serialization/deserialization and infra error wrapping.
- Added `apps/api/src/lib/song-queue-client.ts` with `getSongQueue()` helper and typed client facade.
- Added `SongQueueRpcHandle` in `apps/api/src/durable-objects/song-queue-do.ts` and moved externally used logic behind `*Internal()` methods used by the handle.
- Migrated all non-test `SONG_QUEUE_DO` callers from `getStub("SONG_QUEUE_DO")` to `getSongQueue()`:
  - `routes/api.ts`
  - `routes/stats.ts`
  - `routes/admin.ts`
  - `routes/webhooks.ts`
  - `durable-objects/song-request-saga-do.ts`
- Added/updated regression tests to exercise the new client facade and agent-name bootstrap path.
- Updated the design doc with the practical `connectRpc()` naming note because `DurableObjectStub.connect()` collides with Fetcher socket `connect()`.

Validation:
- `pnpm --filter cf-twitch-api typecheck`
- `pnpm --filter cf-twitch-api test`

Note:
- The concrete control-plane method is `connectRpc(name)` rather than `connect(name)` due the Cloudflare `Fetcher.connect()` collision.
