Implemented `KeyboardRaffleSagaDO` migration to Agent and added RGR-style integration coverage.

Completed:
- Migrated `apps/api/src/durable-objects/keyboard-raffle-saga-do.ts` from `DurableObject<Env>` to `Agent<Env, KeyboardRaffleSagaAgentState>`.
- Added `initialState`, `onStart()`, legacy alarm cleanup, and retry schedule restoration via Agent scheduling.
- Replaced alarm-based retry orchestration with Agent-native one-shot scheduling and public `retrySagaTick()` callback.
- Preserved public RPC surface: `start(params)` and `getStatus()`.
- Kept SQLite/Drizzle as durable source of truth for saga rows/steps; Agent state only coordinates retry schedule id/due time.
- Added integration tests in `apps/api/src/__tests__/durable-objects/keyboard-raffle-saga-do.test.ts` covering:
  1. pre-start `getStatus()` returns `null`
  2. scheduled retry callback resumes a pending retry and completes the saga
  3. startup restores pending retry schedules from durable rows

Validation run:
- `cd apps/api && pnpm exec vitest run src/__tests__/durable-objects/keyboard-raffle-saga-do.test.ts`
- `cd apps/api && pnpm run typecheck`
