## Known blockers to address

### 1) Failing tests: `apps/api/src/__tests__/durable-objects/event-bus-do.schema.test.ts`
There are **4 failures** in the `RaffleRollEvent` tests.

Observed issue:
- The tests construct `RaffleRollEvent` objects **without `isNewRecord`**.
- The current runtime/schema now requires `isNewRecord`.

Likely action:
- Update the tests to match the current schema, or if the schema is wrong, update the schema/factory consistently.
- Verify factory functions and discriminated union tests all align with the final intended event shape.

### 2) Failing tests: `apps/api/src/__tests__/durable-objects/song-request-saga-do.test.ts`
There are **3 failures**.

Observed issues during audit:
- Pending/unused Twitch chat interceptors in tests
- EventBus → Achievements path failed in worker-pool tests with:
  - `no such table: event_history`

Important context:
- This may be partly due to Agent migration test-harness behavior, but it still blocks a deploy decision because the verification suite is red.
- The failing scenario involved `SongRequestSagaDO` / `EventBusDO` / `AchievementsDO` interaction after the Agent migration.

Things to investigate:
- Whether the test needs to explicitly initialize/name any Agent stubs before state/scheduling is exercised.
- Whether the test should seed or initialize `AchievementsDO` differently before EventBus publishes to it.
- Whether the test is over-mocking Twitch chat requests (unused interceptor remains pending).
- Whether the EventBus test path should assert retry behavior differently if `AchievementsDO` isn’t initialized in that harness path.
