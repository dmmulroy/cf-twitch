Migrated `apps/api/src/durable-objects/song-request-saga-do.ts` from `DurableObject<Env>` to `Agent<Env, SongRequestSagaAgentState>` using RGR-style TDD.

Completed work:
- Added new integration tests in `apps/api/src/__tests__/durable-objects/song-request-saga-do.test.ts` covering:
  - initial null status
  - retry scheduling on retryable Spotify failures
  - scheduled resume via `retrySagaTick()`
  - startup restoration of pending retry schedules from saga rows
  - invalid Spotify URL failure semantics
- Refactored `apps/api/src/lib/saga-runner.ts` to use an injected retry scheduler instead of hardcoded DO alarms.
- Updated `apps/api/src/durable-objects/song-request-saga-do.ts` to:
  - extend `Agent`

  - clear legacy alarms on startup
  - coordinate retry scheduling via Agent state (`retryScheduleId`, `retryDueAt`)
  - replace `alarm()` with Agent-native `retrySagaTick()`
  - restore retry schedules from durable saga step rows on startup
- Updated `apps/api/src/durable-objects/keyboard-raffle-saga-do.ts` to pass an alarm-backed retry scheduler into `SagaRunner` so existing behavior remains intact.
- Added saga DO bindings/migrations to `apps/api/wrangler.test.jsonc` for worker-pool tests.
