Implemented focused Agent API improvements for `AchievementsDO` without adding out-of-scope real-time/chat connection features.

Changes:
- moved unlock side effects to Agent background work via `queue("processAchievementUnlockEffects", ...)`
- added `processAchievementUnlockEffects()` background/scheduled callback
- added short announcement retry schedule with delays `[3, 5, 10]` seconds for retryable token/chat preflight failures
- kept durable achievement writes synchronous and behavior-preserving
- retained existing mark-before-send announcement semantics once token preflight succeeds
