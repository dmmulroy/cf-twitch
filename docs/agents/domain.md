# Domain Docs

This repository uses a single-context domain-documentation layout.

## Before exploring

Read:

- `CONTEXT.md` at the repository root
- Relevant ADRs under `docs/adr/`

If either is absent, proceed silently.

## Vocabulary

Use the domain terminology defined in `CONTEXT.md`. Avoid synonyms that its glossary explicitly rejects.

If a required concept is missing, reconsider whether new terminology is necessary or note the gap for domain modeling.

## Architecture decisions

Surface conflicts with existing ADRs explicitly rather than silently overriding them.
